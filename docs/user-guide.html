Pipeliner User Guide

Pipeliner is a media-automation daemon that pulls entries from RSS feeds, torrent indexers, or local filesystems, enriches them with metadata, applies configurable filters, and hands accepted entries off to download clients, notification services, or shell commands.

It is heavily inspired by FlexGet. Pipelines are described in YAML. Each task chains a sequence of plugins. The scheduler runs tasks on cron or interval schedules, persisting state in a SQLite database so each entry is processed only once.

Key concepts

  • Entry — a title, URL, state (undecided/accepted/rejected/failed), and a key-value field map
  • Plugin — a self-contained unit that produces, annotates, filters, modifies, or acts on entries
  • Task — an ordered chain of plugins; metainfo, filter, and modify plugins run in config-file order
  • Phase — the execution category a plugin belongs to (input, metainfo, filter, modify, output, learn)

Quick Start

Create config.yml:

tasks:
  breaking-bad:
    rss:
      url: "https://example.com/rss"
    seen:
    series:
      shows:
        - "Breaking Bad"
    transmission:
      host: localhost
      port: 9091

schedules:
  breaking-bad: 1h

Run once to test:

pipeliner run --config config.yml

Run as a daemon (with web UI):

pipeliner daemon --config config.yml --web :8080 \
  --web-user admin --web-password secret

Validate config without running:

pipeliner check --config config.yml

Configuration File

Pipeliner uses a single YAML file (default: config.yml). Pass a custom path with --config.

Top-level structure

variables:          # substitution values applied before parsing
  tv_root: /media/tv

templates:          # named partial configs that tasks can inherit
  base:
    seen:

tasks:              # one or more named pipelines
  my-task:
    template: base
    rss:
      url: "https://example.com/rss"

schedules:          # when to run each task in daemon mode
  my-task: 1h

Database

Pipeliner automatically maintains pipeliner.db (SQLite) in the same directory as the config file. All stateful plugins (seen, series, movies, premiere, upgrade, the TVDB/Trakt caches, and persistent lists) share this file. No configuration needed.

Variables

Two substitution syntaxes are applied to the raw YAML bytes before parsing:

SyntaxSourceExample
{$ key $}variables: block in config{$ tv_root $}
${ENV_VAR}Process environment${TVDB_API_KEY}
variables:
  tv_root: /media/tv
  tvdb_key: YOUR_KEY

tasks:
  tv:
    metainfo_tvdb:
      api_key: "{$ tvdb_key $}"
    pathfmt:
      path: "{$ tv_root $}/{series_name}"
Substituted values are injected into raw YAML before parsing. Values containing YAML metacharacters (colons, newlines) can alter config structure. Only substitute values from trusted sources.

Templates

Templates let you define shared plugin config once and merge it into multiple tasks. A task references a template with template: name. Per-task plugin config is shallowly merged on top (task wins on key conflicts).

templates:
  tv-base:
    jackett_input:
      url: "https://jackett.example.com"
      api_key: "${JACKETT_KEY}"
    torrent_alive:
      min_seeds: 1
    deluge:
      host: localhost
      password: changeme
    email:                   # smtp settings shared; task adds subject/body
      smtp_host: smtp.gmail.com
      smtp_port: 587
      from: me@example.com
      to: me@example.com

tasks:
  tv-premiere:
    template: tv-base
    premiere:
      quality: 720p+ webrip+
    email:                   # merged with template's smtp fields
      subject: "New premiere: {{len .Entries}} show(s)"

Merge rules

  • Template plugins come first in the merged order; task-only plugins are appended.
  • When both template and task define the same plugin, their configs are shallowly merged (task fields win on conflict). This lets a task supply just subject and body_template while inheriting smtp settings from a template.
  • Append ! to a plugin key in the task (e.g. email!) to completely replace the template value instead of merging.
  • Multiple templates: template: [base, extras] — merged left to right, then task on top.

Schedules

In daemon mode, tasks run automatically according to their schedule. Both interval strings and 5-field cron expressions are supported.

schedules:
  hourly-task:  1h
  daily-task:   24h
  morning-task: "0 7 * * *"   # every day at 07:00

Tasks without a schedule entry are not run automatically; trigger them manually via the web UI or pipeliner run.

Pipeline Phases

All input plugins run first (concurrently), then metainfo, filter, and modify plugins run in the order they appear in the YAML, and finally all output plugins run (concurrently). The learn phase follows output.

PhaseWhat happensEntry state after
InputPlugins produce entries from sources (RSS, files, searches). All inputs run concurrently; results are merged and deduplicated by URL.undecided

OutputPlugins act on all accepted entries (add to torrent client, send email). All outputs run concurrently.accepted / failed (if downloader errors)
LearnPlugins persist state for future runs (mark as seen, record series tracker). Skipped in dry-run mode.—
Metainfo, filter, and modify plugins run in config-file order. Place a filter immediately after the metainfo plugin that sets the field it inspects to reject cheap early and avoid unnecessary annotation work for entries that will be rejected anyway.

Input Plugins

Input plugins produce entries. All inputs in a task run concurrently; their results are merged and deduplicated by URL before other phases begin.

rss

Fetches entries from an RSS 2.0 or Atom 1.0 feed. Prefers enclosure URLs (torrent feeds) and falls back to item links.

KeyRequiredDefaultDescription
urlyes—Feed URL
all_entriesnofalseSkip filter phase; accept all entries

Fields set

rss_feed, rss_link, rss_description, rss_pubdate, rss_guid, rss_enclosure_url, rss_enclosure_type, torrent_seeders (when present in feed)

rss:
  url: "https://example.com/torrents/rss"

html

Scrapes all <a href> links from an HTML page. Optionally filters by glob pattern.

KeyRequiredDefaultDescription
urlyes—Page URL to fetch
maskno—Glob filter on link filenames (e.g. *.torrent)
html:
  url: "https://example.com/downloads"
  mask: "*.torrent"

filesystem

Walks a local directory and emits one entry per file with a file:// URL.

KeyRequiredDefaultDescription
pathyes—Directory to scan
recursivenofalseRecurse into subdirectories
maskno—Glob filter (e.g. *.torrent)

Fields set

location (absolute path), filename, extension, size, modified_time

filesystem:
  path: /downloads/watch
  mask: "*.torrent"

discover

Actively searches multiple sources for entries matching a title list. Unlike RSS (which passively receives all items), discover iterates titles, dispatches one search query per title to each configured search plugin (via), and returns the merged results. A per-title cooldown prevents redundant searches.

KeyRequiredDefaultDescription
titlesconditional—Static list of title strings
fromconditional—From-plugin configs whose entry titles supplement the list
viayes—Search plugin configs to query
intervalno24hMinimum time between searches per title

At least one of titles or from is required.

discover:
  from:
    - name: trakt_list
      client_id: YOUR_CLIENT_ID
      access_token: YOUR_TOKEN
      type: movies
      list: watchlist
  via:
    - name: jackett
      url: "http://localhost:9117"
      api_key: YOUR_JACKETT_KEY
      categories: ["2000"]
  interval: 12h

jackett_input

Returns recent results from Jackett indexers without a search query — behaves like an RSS feed. Queries the configured categories and returns whatever the indexers consider recent.

KeyRequiredDefaultDescription
urlyes—Jackett base URL
api_keyyes—Jackett API key
indexersno["all"]Indexer IDs to query
categoriesno—Torznab category codes (e.g. ["5040","5045"])
limitno—Max results across all indexers
timeoutno60sHTTP request timeout

Fields set

torrent_seeders, torrent_leechers, torrent_size, torrent_info_hash, jackett_category, jackett_indexer

Common Torznab categories

From Plugins

From plugins are sub-plugins — they cannot be used as top-level task plugins and will cause a validation error if you try. They are referenced inside series.from, movies.from, discover.from, or discover.via.

series:
  from:
    - name: tvdb_favorites   # from plugin with inline config
      api_key: YOUR_KEY
      user_pin: YOUR_PIN
    - trakt_list            # from plugin by name alone (uses defaults)

tvdb_favorites

Fetches a user's TheTVDB favorites list and emits one entry per show. Use inside series.from or discover.from to build a dynamic show list from your TVDB account.

KeyRequiredDescription
api_keyyesTheTVDB v4 API key
user_pinyesUser PIN from thetvdb.com account settings

Get your API key and user PIN at thetvdb.com/api-information.

series:
  tracking: backfill
  quality: 720p+ webrip+
  from:
    - name: tvdb_favorites
      api_key: YOUR_TVDB_API_KEY
      user_pin: YOUR_TVDB_USER_PIN

trakt_list

Fetches movies or shows from a Trakt.tv list. Use inside series.from, movies.from, or discover.from.

KeyRequiredDefaultDescription
client_idyes—Trakt API Client ID
typeyes—movies or shows
listnowatchlistwatchlist, trending, popular, watched, ratings, collection
access_tokenconditional—Required for watchlist, ratings, collection
limitno100Max results for public lists

Get a free API key at trakt.tv/oauth/applications.

jackett

Searches a Jackett indexer proxy via Torznab. Used inside discover.via as a search backend. Seeder counts, info hashes, and sizes come directly in the search response — no separate metainfo fetch needed.

KeyRequiredDefaultDescription
urlyes—Jackett base URL
api_keyyes—Jackett API key
indexersno["all"]Indexer IDs
categoriesno—Torznab category codes
limitno—Max results
timeoutno60sHTTP timeout

rss_search

Fetches entries from a parameterized RSS URL. Used inside discover.via. The URL template receives the search query.

KeyRequiredDescription
url_templateyesGo template for the search URL

Template variables: {{.Query}} (raw) and {{.QueryEscaped}} (URL-encoded). Brace syntax {QueryEscaped} also works.

discover:
  titles: ["Breaking Bad"]
  via:
    - name: rss_search
      url_template: "https://jackett.example.com/api?q={{.QueryEscaped}}&apikey=KEY"
  interval: 6h

Metainfo Plugins

Metainfo plugins annotate entries with additional fields. They run in config-file order, interleaved with filter and modify plugins — place a metainfo plugin immediately before the filter that depends on its output.

metainfo_tvdb

Enriches series entries with TheTVDB metadata. Only processes entries whose title parses as a series episode. Results are cached in pipeliner.db.

KeyRequiredDefaultDescription
api_keyyes—TheTVDB v4 API key
cache_ttlno24hHow long to cache results

Series-level fields set: tvdb_id, tvdb_series_name, tvdb_series_year, tvdb_overview, tvdb_slug, tvdb_network, tvdb_language, tvdb_country, tvdb_genres, tvdb_poster, tvdb_first_air_date, tvdb_last_air_date, tvdb_next_air_date, tvdb_status, tvdb_content_rating, tvdb_trailers, tvdb_score, tvdb_aliases, tvdb_cast

Episode-level fields (when S/E is parsed): tvdb_episode_id, tvdb_episode_name, tvdb_air_date, tvdb_episode_overview, tvdb_episode_runtime, tvdb_episode_image

metainfo_tmdb

Enriches movie entries with TMDb metadata. Only processes entries whose title parses as a movie (title + year). Results cached in pipeliner.db.

KeyRequiredDefaultDescription
api_keyyes—TMDb API v3 key
cache_ttlno24hHow long to cache results

Fields set: tmdb_id, tmdb_title, tmdb_original_title, tmdb_release_date, tmdb_overview, tmdb_popularity, tmdb_vote_average, tmdb_runtime, tmdb_tagline, tmdb_imdb_id, tmdb_genres

Get a free key at themoviedb.org/settings/api.

metainfo_trakt

Annotates entries with Trakt.tv metadata. Results cached in pipeliner.db.

KeyRequiredDefaultDescription
client_idyes—Trakt API Client ID
typeyes—shows or movies
cache_ttlno24hHow long to cache results

Fields set: trakt_id, trakt_slug, trakt_imdb_id, trakt_tmdb_id, trakt_tvdb_id, trakt_title, trakt_year, trakt_overview, trakt_rating, trakt_votes, trakt_genres

metainfo_quality

Parses video quality tags from the entry title. Takes no config.

Fields set: quality, resolution, source, codec, audio, color_range

metainfo_quality:    # no config needed

metainfo_series

Parses series and episode information from the entry title. Takes no config. (The series filter also does this internally — use this standalone plugin when you need the fields without the series filter's show-list matching.)

Fields set: series_name, series_season, series_episode, series_episode_id, series_proper, series_repack, series_service

metainfo_torrent

Reads metadata from a .torrent file. Reads from location field (filesystem plugin) or downloads the URL if it ends in .torrent. Takes no config.

Fields set: torrent_name, torrent_info_hash, torrent_size, torrent_file_count, torrent_announce, torrent_comment, torrent_private

metainfo_magnet

Annotates magnet-link entries. URI-derived fields are set immediately; full metadata (name, size, file list) is resolved via DHT. All DHT lookups in a batch run in parallel sharing a single timeout window. Non-magnet entries are skipped.

KeyRequiredDefaultDescription
resolve_timeoutno30sTotal DHT resolution timeout for the entire batch

From URI: torrent_info_hash, torrent_announce, torrent_announce_list, torrent_display_name
After DHT: torrent_name, torrent_size, torrent_file_count, torrent_files

Filter Plugins

Filter plugins run in YAML config order, interleaved with metainfo and modify plugins. Each entry is passed to every filter in sequence; once an entry is rejected or failed it is skipped by subsequent filter plugins.

series

Accepts episodes of configured TV shows. Parses the episode identifier from the title, matches series names with fuzzy matching, and enforces optional quality and ordering constraints. Multiple quality variants of the same episode are all accepted — the task engine's automatic deduplication picks the best copy. Download history is recorded in the Learn phase.

KeyRequiredDefaultDescription
showsconditional—Static list of show names
fromconditional—From-plugin configs providing show names dynamically
ttlno1hCache TTL for dynamic show list
trackingnostrictEpisode ordering mode: strict, backfill, or follow
qualityno—Minimum quality spec (e.g. 720p+ webrip+)

Tracking modes

ModeBehaviour
strictAccept only the next expected episode; reject gaps >1 ahead of latest downloaded. Does not handle full-season binge drops well.
backfillAccept any episode not yet downloaded, including older ones. Downloads full back-catalogue.
followRecommended for new and continuing series. On first encounter accept everything (full season drops in one pass). Afterwards the earliest tracked season acts as an anchor — episodes from older seasons are rejected, all episodes in the anchor season or newer are accepted (including gap-fills within the current season).

Fields set: series_name, series_season, series_episode, series_episode_id

series:
  tracking: strict
  quality: 720p+ webrip+
  shows:
    - "Breaking Bad"
    - "Better Call Saul"
# Dynamic list from TVDB favorites — backfill all episodes not yet downloaded
series:
  tracking: backfill
  quality: 720p+ webrip+
  from:
    - name: tvdb_favorites
      api_key: YOUR_KEY
      user_pin: YOUR_PIN

movies

Accepts movies from a configured title list. Same quality and deduplication behavior as series. Download history recorded in Learn phase.

KeyRequiredDefaultDescription
moviesconditional—Static list of movie titles
fromconditional—From-plugin configs providing titles dynamically
ttlno1hCache TTL for dynamic list
qualityno—Minimum quality floor

Fields set: movie_title, movie_year, movie_quality, movie_3d

Use condition: reject: 'movie_3d == true' to exclude 3D releases. Detected markers include: 3D, SBS, HSBS, HOU, BD3D, and variants.

premiere

Accepts premiere episodes of previously unseen series. Once a premiere is downloaded, subsequent runs reject that series. Useful for discovering new shows; let series take over episode tracking after the first download.

KeyRequiredDefaultDescription
episodeno1Episode number to treat as premiere
seasonno1Season number to match; 0 means any
qualityno—Quality spec the entry must satisfy
premiere:
  quality: 720p+ webrip+

condition

Accepts or rejects entries using boolean expressions. See the Expression Language section for full syntax.

KeyRequiredDescription
acceptconditionalAccept when expression is true
rejectconditionalReject when expression is true
rulesconditionalOrdered list of {accept, reject} objects (use when you need multiple conditions)
condition:
  rules:
    - reject: 'tvdb_language != "" and tvdb_language != "English"'
    - reject: 'tvdb_genres contains "Documentary"'
    - reject: 'tvdb_genres contains "Reality"'
    - reject: 'tvdb_first_air_date != "" and tvdb_first_air_date < daysago(365)'
    - accept: 'tmdb_vote_average >= 7.0'

torrent_alive

Rejects torrents with fewer than a minimum number of seeds. The torrent_seeders field is set by the RSS input, Jackett, and jackett_input. Entries without seed data are left undecided.

KeyRequiredDefaultDescription
min_seedsno1Minimum seed count required
torrent_alive:
  min_seeds: 3

require

Rejects entries missing one or more required fields. A field is "missing" if its value is nil, empty string, zero, false, or empty slice.

require:
  fields:
    - tvdb_id
    - series_name

seen

Rejects entries already processed in a previous run, using a SHA-256 fingerprint of configured fields stored in pipeliner.db. Fingerprints are written in the Learn phase — if an output plugin fails, the entry is not marked seen and will be retried.

Not needed for TV series, movies, or premieres. Those filters maintain their own download trackers and handle deduplication automatically. Use seen for pipelines without a domain-specific tracker — the most common case is RSS news or article feeds, where the same article would otherwise be re-processed every run.
KeyRequiredDefaultDescription
fieldsno["url"]Fields to include in the fingerprint
localnofalseScope seen store to this task name only

Typical use — tech news to email, preventing re-delivery of already-sent articles:

tasks:
  tech-news:
    rss:
      url: "https://feeds.arstechnica.com/arstechnica/technology-lab"
    seen:                  # reject already-sent articles by URL
    regexp:
      accept: "(?i)linux|golang|open.?source"
    email:
      smtp_host: smtp.gmail.com
      subject: "{{len .Entries}} new article(s)"

schedules:
  tech-news: 1h

tvdb

Accepts series entries whose parsed show name fuzzy-matches a show in the user's TheTVDB favorites. Fetches and caches the favorites list.

KeyRequiredDefaultDescription
api_keyyes—TheTVDB API key
user_pinyes—User PIN from thetvdb.com
ttlno1hFavorites list cache TTL

trakt

Accepts entries whose parsed title matches something on a Trakt.tv list.

KeyRequiredDefaultDescription
client_idyes—Trakt API Client ID
typeyes—shows or movies
listnowatchlistList name
access_tokenconditional—Required for private lists
limitno100Max results for public lists
min_ratingno—Minimum rating (1–10, ratings list only)

Output Plugins

Output plugins act on accepted entries after modify. All outputs in a task run concurrently.

Error handling: Downloader plugins (deluge, qbittorrent, transmission, download) mark an entry as failed when they cannot queue or fetch it. Failed entries are skipped by the Learn phase and will be retried on the next run. Notification plugins (email, exec, print, notify) do not mark entries failed — a notification failure does not cause a retry.

Web UI

Start the web UI with pipeliner daemon --web :8080 --web-user admin --web-password secret. Navigate to http://localhost:8080.

The interface has three tabs: Dashboard, Config, and Database. A User Guide link in the header opens this document.

Dashboard tab

  • Task cards — each card shows the next scheduled run datetime (hover for the raw cron/interval), the last run time, accepted/rejected/failed counts, and a Run now button.
  • Run all — triggers all tasks immediately.
  • Reload config — reloads config.yml from disk without restarting the process. Queued for after running tasks finish if any are active.
  • Live Log — SSE-streamed log with a filter box. Reconnects automatically; replays the last 10,000 lines for new connections.

Config tab

A live YAML editor with syntax highlighting. Use Validate to check config without saving, or Save & Reload to write and apply immediately (or queue a reload if tasks are running). Tab key inserts two spaces.

Database tab

Inspect and manage the two download trackers stored in pipeliner.db.

A sidebar lists the available trackers; click one to load its contents in the main panel. A filter box and Clear all button appear for the active tracker.

  • Series — all TV episodes recorded as downloaded, grouped by show. Expand a show to see each episode's ID, quality, and download date. Delete an individual episode or an entire show to force a re-download on the next run. Includes entries from both the series and premiere filters.
  • Movies — all movies recorded as downloaded. Delete a movie to force a re-download.
  • Seen — entries recorded by the seen filter, showing title, task name, and date. Delete an entry to allow it to be processed again. Per-task local seen stores (local: true) appear as separate sidebar entries named Seen: task-name.

HTTPS

Pass --tls-cert and --tls-key for a real certificate, or --tls-self-signed to generate one automatically. With TLS the Secure cookie flag is set automatically.

Dry-Run Mode

Pass --dry-run to pipeliner run to execute the full pipeline (input → processing → output skipped) but skip the output and learn phases. The run is fully idempotent — no torrents are queued, no emails sent, no state written to pipeliner.db. You can run it repeatedly to inspect what would be accepted without side effects.

pipeliner run --config config.yml --dry-run --log-level debug

Troubleshooting

No entries accepted

  • Run with --log-level debug to see why each entry is rejected. The reject reason is logged for every rejected entry.
  • Add print: before output plugins to confirm entries reach the output phase.
  • Check --dry-run output — if entries are accepted in dry-run but not live, the output plugin may be failing.

TVDB not finding a series

  • The series name parser is strict. Add --log-level debug --log-plugin metainfo_tvdb to see what name it searches for.
  • The TVDB search uses the parsed series name — if the release title uses an unusual name variant, TVDB may return the wrong result or nothing.
  • Check that your API key and user PIN are correct at thetvdb.com/api-information.

Deluge "Not authenticated" errors

This was a bug where the Deluge session cookie from login was not preserved. It is fixed in v0.1.3+. Ensure you are running a current version.

torrent_alive is slow

When entries lack a torrent_seeders field, torrent_alive must resolve the info hash and scrape trackers. Use inputs that provide seed counts (Jackett, jackett_input, nyaa RSS) to avoid scraping. Set scrape: false to disable tracker scraping entirely (entries without seed data will be left undecided).

Config validation errors from templates

pipeliner check validates each plugin against the fully merged config (template + task), so required fields supplied by a template are correctly recognized. If you see "required field missing" errors for fields you expect to come from a template, ensure the template name is spelled correctly and the plugin key matches.

Schedule not running

Tasks without a schedules: entry are not run automatically by the daemon. Add a schedule or trigger manually with pipeliner run task-name or the web UI's Run now button.

PhaseFrom plugin used as top-level input

If you see "plugin X is a from plugin; use it via series.from, movies.from, or discover.from instead", move the plugin inside a from: list with inline config:

# Wrong — top-level task plugin:
tvdb_favorites:
  api_key: YOUR_KEY

# Correct — inside series.from:
series:
  from:
    - name: tvdb_favorites
      api_key: YOUR_KEY
      user_pin: YOUR_PIN
